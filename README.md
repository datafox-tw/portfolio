Rather than showcasing isolated code snippets, this GitHub is organized to reflect how I think, design, and iterate on problems.

# How to read this GitHub

My public repositories fall roughly into the following categories:

1. Finance × CS core projects
These repositories best represent how I combine financial intuition with engineering structure.
They emphasize problem framing, system boundaries, and design decisions over surface-level performance metrics.

2. Course projects (selected)
While some repositories originated as course assignments, the ones listed below were refactored or selected because they contain reusable structure, non-trivial insights, or clear experimental logic.

3. Data & reference contributions
In addition to projects, I maintain repositories that provide useful datasets, notes, or structured references intended to support analysis or future work.

Not every repository is meant to be read end-to-end.
For a focused review, I recommend starting with the selected projects below.
